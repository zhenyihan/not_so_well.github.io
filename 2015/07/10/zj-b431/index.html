Problem

背景

中國餘數定理 (Chinese Remainder Theorem，簡稱 CRT) 經常是工程學裡面常用的一種轉換域，很多人不知道當初在大學離散數學中學這個做什麼，但是在不少計算的設計都會運用到 CRT。由於電腦 CPU 架構中的運算單位是 32-bits 或者 64-bits (也許在未來會更長)，但值域高達 128-bits 或者 512-bits 以上模擬運算成了麻煩之處。

問題描述

來個簡單運用，來計算簡單的 RSA 加解密，特化其中的數學運算。

Solution

RSA 可以預先處理

  • cp=q×(q−1\modp)c_p = q \times (q^{-1} \mod p)c​p​​=q×(q​−1​​\modp)
  • cq=p×(p−1\modq)c_q = p \times (p^{-1} \mod q)c​q​​=p×(p​−1​​\modq)

還原的算法則是 M=Mp×cp+Mq×cq\modNM = Mp \times c_p + Mq \times c_q \mod NM=Mp×c​p​​+Mq×c​q​​\modN

由於拆分後的 bit length 少一半，乘法速度快 4 倍，快速冪次方快 2 倍 (次方的 bit length 少一半)，但是要算 2 次，最後共計快 4 倍。CPU 的乘法想必不會用快速傅立葉 FFT 來達到乘法速度為 O(nlogn)O(n \log n)O(nlogn)。

特別小心「 bit length 少一半 」必須在 gcd(C,p)=1gcd(C, p) = 1gcd(C,p)=1 時才成立，互質機率機率非常高，仍然有不成立的時候，這情況下速度不是加快 400%。請參照一般解。

例如 C = 27522, d = 17132, p = 2, q = 17293，若使用歐拉定理計算 Mp=Cd\mod(p−1)\modp=1Mp = C^{d \mod (p-1)} \mod p = 1Mp=C​d\mod(p−1)​​\modp=1 事實上 Mp=Cd\modp=0Mp = C^d \mod p = 0Mp=C​d​​\modp=0。有一個特性尚未被利用，對於模質數 ppp 而言，所有數 xxx 在模 ppp 下的循環長度 L∣(p−1)L | (p-1)L∣(p−1)，最後可以套用 Mp=Cd\mod(p−1)+(p−1)\modpMp = C^{d \mod (p-1) + (p-1)} \mod pMp=C​d\mod(p−1)+(p−1)​​\modp 來完成。請參照循環解，如此一來就不必先判斷 gcd(C,p)=1gcd(C, p) = 1gcd(C,p)=1。

番外

但是利用 CRT 計算容易受到硬體上攻擊，因為會造成 p,qp, qp,q 在分解過程中獨立出現，當初利用 NNN 很難被分解的特性來達到資訊安全，但是卻因為加速把 p,qp, qp,q 存放道不同時刻的暫存器中。

其中一種攻擊，計算得到 M=Mp×q×(q−1\modp)+Mq×p×(p−1\modq)\modNM = Mp \times q \times (q^{-1} \mod p) + Mq \times p \times (p^{-1} \mod q) \mod NM=Mp×q×(q​−1​​\modp)+Mq×p×(p​−1​​\modq)\modN 當擾亂後面的式子 (提供不穩定的計算結果)。得到 M′=Mp×q×(q−1\modp)+(Mq+Δ)×p×(p−1\modq)\modNM' = Mp \times q \times (q^{-1} \mod p) + (Mq + \Delta) \times p \times (p^{-1} \mod q) \mod NM​′​​=Mp×q×(q​−1​​\modp)+(Mq+Δ)×p×(p​−1​​\modq)\modN

接著 (M′−M)=(Δ′×p)\modN(M' - M) = (\Delta' \times p) \mod N(M​′​​−M)=(Δ​′​​×p)\modN，若要求 ppp 的方法為 gcd(M′−M,N)=gcd(Δ′×p,N)=pgcd(M' - M, N) = gcd(\Delta' \times p, N) = pgcd(M​′​​−M,N)=gcd(Δ​′​​×p,N)=p，輾轉相除的概念呼之欲出，原來 ppp 會被這麼夾出，當得到兩個 p,qp, qp,q，RSA 算法就會被破解。
